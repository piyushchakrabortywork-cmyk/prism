Documentation

Everything you need to integrate PRISM into your workflow.

PRISM is a developer-first tool designed to bring intelligent code review capabilities directly into your pull requests. It catches bugs, security vulnerabilities, and architectural issues before they merge.

Getting Started

To get started with PRISM, you'll need a GitHub account and a repository where you have admin permissions.

Installation

You can install the PRISM GitHub App directly from the marketplace.

bash 
npx prism-cli init

Configuration

Create a .prism.yml file in the root of your repository to configure the analysis rules.

yaml 
rules:
  security: strict
  performance: warning
  complexity_threshold: 12
 
ignore:
  - "dist/**"
  - "**/*.test.ts"
